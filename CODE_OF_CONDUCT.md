# Code of Conduct

## Our Commitment

We want this project to be a respectful, inclusive, and constructive space for all contributors.

## Expected Behavior

- Be respectful and professional in issues, pull requests, and discussions.
- Give actionable feedback focused on code and content, not people.
- Assume positive intent and ask clarifying questions before escalating.
- Keep discussions on topic and related to project goals.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Personal attacks, trolling, or hostile interactions.
- Sharing private or sensitive information without consent.
- Repeatedly ignoring project conventions after review feedback.

## Scope

This Code of Conduct applies in all project spaces, including issues, pull requests, discussions, and review comments.

## Enforcement

Maintainers may edit, hide, or remove content that violates this Code of Conduct and may limit participation when needed.

## Reporting

If you experience or witness unacceptable behavior, open a private maintainer contact via GitHub (or use private vulnerability/contact channels if enabled).

## Project-Specific Note

Respect contributor time by following the documented workflow and templates before opening a pull request.
